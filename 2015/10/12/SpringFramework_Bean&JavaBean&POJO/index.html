Spring Bean

A Spring bean is basically an object managed by Spring.
More specifically, it is an object that is instantiated, configured and otherwise managed by a Spring Framework container.

在Spring中，Bean是一個物件，藉由Spring IoC Container來初始化(instantiated)、管理組裝(assembled)。

我的理解是他是spring new 出來的物件

一個典型Bean 的.xml設定檔如下所示：

<beans xmlns="http://www.springframework.org/schema/beans"
xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
xsi:schemaLocation="http://www.springframework.org/schema/beans
    http://www.springframework.org/schema/beans/spring-beans-3.0.xsd">
   <bean id="helloWorld" class="HelloWorld" destroy-method="destroy" init-method="init">
       <property name="message" value="Hello World!"/>
   </bean>
</beans>

JavaBean

  • Have a public default (no argument) constructor

  • allows access to properties using accessor (getter & setter) methods

  • Implement java.io.Serializable

標準的Java Beans必須符合以下條件:

  1. 為一個 public 類別
  2. 實作 java.io.Serializable 介面
  3. 必須有一個沒有傳入參數的建構子
  4. Properties 均有一對 getter & setter method

example:

//1.為一個 public 類別
//2.實作 java.io.Serializable 介面 
public class User implements java.io.Serializable {
    private String name;
    private Integer age;

    //3.必須有一個沒有傳入參數的建構子
    public User(){

    }

    //4.Properties 均有一對 getter & setter method
    public String getName(){
        return this.name;
    }

    public void setName(String name){
        this.name = name;
    }

    public Integer getAge(){
        return this.age;
    }

    public void setAge(Integer age){
        this.age = age;
    }
}

Plan Old Java Project（POJO）

POJO是一個正規的Java物件(不是JavaBean、EntityBean等)，也不擔當任何的特殊的角色，

也不實現任何Java框架指定的接口。

所以POJO與JavaBean是不同的，因為POJO它不可依賴於framework的繼承或interface實作。

與JavaBeans不同的是:POJO包含業務邏輯處理和持久化邏輯；

與JavaBeans相同的是:可以包含屬性的set和get方法。

example:

public class MyBean {

    private String someProperty;

    public String getSomeProperty() {
         return someProperty;
    }

    public void setSomeProperty(String someProperty) {
        this.someProperty = someProperty;
    }
}
