Factory Mode

發表於 2015-10-17   |   分類於 DesignPattern ， Facotry

目的

定義 interface , 讓 繼承的類別來決定哪些class需要被實體化

簡單來說就是把實體化的動作由繼承類別來實作

透過Factory 來統一管理 所有interfce的實作

之後就可以透過Facotry 直接操作。

例子

比方說我們有一個畫髮型的Interface

public interface HairInterface{
    public void draw();
}

因為髮型有分往左旁分的或著往右旁分，所以我們的實作就會分成兩支

往左旁分的程式

public class LeftHair implements HairInterface {
    @override
    public void draw(){
        System.out.println("往左旁分");
    }
}

往右旁分的

public class RightHair implements HairInterface {
    @override
    public void draw(){
        System.out.println("往右旁分");
    }
}

當我們在使用時,希望往左旁分就會如下

HairInterface left = new LeftHair();
left.draw();

但這樣的寫法不方便，故我們會做一個Facotry class來幫我統一管理這些實作

public class HairFactory {

    public HairInterface getHairByClass(String className){

        try{
            HairInterface hair = (HairInterface) Class.forName(className).newInstance();
            return hair;
        }catch(Exception e){
            return null;
        }
    }

}

使用時就變成

HairFactory factory = new HairFactory();
HairInterface left = factory.getHairByClass(LeftHair.getClass().getName());
left.draw();

方便我們呼叫使用
