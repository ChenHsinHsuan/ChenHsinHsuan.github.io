MVC

Model View Controller

  • 主要目的在於程式分層,抽離
  • 並提供我們開發時的一種思考方式

Model: 業務資料的裝載容器, 負責整理需要展示資料給User。
View: 提供UI畫面, 負責以優美及合理的的方式展示資料給User。
Controller: 負責操作業務邏輯放置到model並決定傳輸哪些資料到View, 負責使用正確的業務邏輯呈現正確的資料及提升較好的性能。

Spring MVC概念

Front Controller 就是 DispatcherServlet 前端控制器

當user 發出 request 會先接觸 Front Controller (DispatcherServlet)

透過 Front Controller (DispatcherServlet)分發到合適的 Controller 生產出我們要的Model 資料 回到 Front Controllre(DispatcherServlet) 再傳送到指定的View 呈現資料

Controller

使用業務邏輯產生Model資料的地方

HandlerAdapter

DispatcherServlet 內部的一種class

也是controller的一種表現形式，因為沒有一種interface 叫做controller

故透過HandlelrAdapter這個 class, 就可以讓DispatcherServlet 去使用Controller

HandlerInterceptor

是一種interface的攔截器

可於於controller之前或之後使用一些方法

HandlerMapping

DispatcherServlet 與 controller 映射關係的一種class

負責告訴DispatcherServlet 當request來了之後，

由哪一個controller來處理，透過Mapping之後，

DispatcherServlet 就會透過HandlerAdapter知HandlerInterceptor及Controller。

#HandlerExecutionChain

preHandle -> Controller method -> postHandle -> afterCompletion

#ModalAndView

是Modal的具體表現

ViewResolver

根據我們的設定，拉出對應的View

View

負責呈現頁面

整個流程順序

  1. request 先發給 DispatcherServlet
  2. DispatcherServlet 會去找 HandlerMapping
  3. 拉出相關的Controller 以及 HandlerInterceptor
  4. 產生所謂的 HandlerExecutionChain
  5. Controller 和 HandlerInterceptor 會組成一般化的HandlerAdapter
  6. DispatcherServlet 就可以使用 HandlerAdapter
  7. 產生對應的ModelAndView
  8. DispatcherServlet 就會透過ViewResolver
  9. 拉出對應的 View並傳遞Model
  10. response 給User
