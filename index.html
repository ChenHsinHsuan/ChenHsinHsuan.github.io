演算法課程心得(二)

發表於 2015-10-15   |   分類於 網路課程 ， 演算法 ， 心得

#OverView

##Stable Matching Problem

有100個男生和100個女生

每個女生會給你他們每個人對這100個男生的喜好順序
每個男生也會給你他們每個人對這100個女生的喜好順序

希望配對後是 1對1 , 且無重複，且選擇的對象必須是彼此的最高順位，列出演算法

##問題的描述

Perfect maching

每個人都有配到對象

Stability

不存在喜歡的對象比被配對的對象更優

Stable matching

符合上方兩個定義

stable matching problem

##問題的解法

解法一：隨機先配對，最後再修正

  1. start with some matching
  2. while there is an unstable pair do
  3. Swap mates to make the pair stable

步驟2 和 步驟3 有可能會產生無限迴圈
會與 演算法的「在有限步驟內」的原則相牴觸
故不是很好的解法

BeanFactory&ApplicationContext getBean的差異

發表於 2015-10-14   |   分類於 Web ， Java ， framwork ， spring ， 名詞定義

透過BeanFactory getBean

不會自動new 物件,等待getBean時才會new

好處是可以節省記憶體，缺點是速度上可能會有差異

透過ApplicationContext getBean

bean在scope預設為singleton的情況下，會自動的new 出物件

好處是會預先的載入物件 instance，缺點是耗費記憶體

由於設備硬體的成本降低，目前大部分的使用上都已使用ApplicationContext來做bean的存取操作
